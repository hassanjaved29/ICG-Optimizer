1 — Input Expression Stage 1

Enter arithmetic assignments in infix notation, one per line.

2 — Postfix Conversion Stage 2

Each RHS is converted to postfix (Reverse Polish Notation) using a stack. Operators are placed after their operands, removing the need for parentheses or precedence rules at later stages.

3 — Three-Address Code (TAC) Stage 3

Each operation is broken into instructions with at most two operands and one result. Temporary variables (t1, t2, ...) hold intermediate values.

4 — Copy Propagation Stage 4

When a variable is assigned a copy of another (e.g. b = a), later uses of that variable are replaced directly with the original. Redundant copy instructions are then removed.

5 — Common Subexpression Elimination (CSE) Stage 5

If the same expression is computed more than once and its operands haven't changed, the second computation is replaced with the result of the first.
